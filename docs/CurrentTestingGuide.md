# HeartLink: текущая инструкция для теста

## Что изменилось

- На первом экране больше нет поля адреса сервера.
- Приложение показывает только ваш код, поле кода партнера и тестовую кнопку.
- Добавлен сброс пары на главной: `Быстрые действия -> Сброс`.
- Локальный сервер теперь хранит сообщения, воспоминания, цели и настроение.
- IPA workflow больше не удаляет `PlugIns`, поэтому виджетная extension попадает в сборку.

## Как запускать сервер

```powershell
cd C:\Users\dmtiryyr\Documents\Codex\2026-05-21\create-a-modern-ios-app-for\local-server
npm install
npm start
```

Сервер должен быть доступен на:

```text
http://192.168.100.4:3000
```

На iPhone в Safari проверь:

```text
http://192.168.100.4:3000/api/health
```

Если Safari не открывает адрес, разреши порт:

```powershell
New-NetFirewallRule -DisplayName "HeartLink Local Server 3000" -Direction Inbound -Action Allow -Protocol TCP -LocalPort 3000
```

## Как пройти первый запуск

1. Открой HeartLink.
2. Скопируй или запомни личный код.
3. Если второго iPhone нет, нажми `Создать тестового партнера`.
4. Введи свое имя, имя партнера и дату начала отношений.
5. Нажми `Открыть HeartLink`.

## Как начать заново

1. Открой `Главная`.
2. В блоке `Быстрые действия` нажми `Сброс`.
3. Подтверди `Сбросить пару`.
4. Приложение вернется к экрану кода.

## Про виджеты

Виджет теперь добавляется в IPA, но бесплатная установка через Sideloadly может потребовать дополнительную подпись для extension. Если iPhone все равно не показывает HeartLink в списке виджетов, значит Sideloadly не подписал виджетную extension. Это ограничение бесплатной sideload-схемы, не SwiftUI-кода.
